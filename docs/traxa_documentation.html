1. نظرة عامة على المشروع

1.1 الفكرة

Traxa هو تطبيق Flutter للهواتف يدير المنظومة الأكاديمية لكلية: الحضور (Face Detection + QR)، الدرجات (GPA متراكم)، الجداول، إدارة المعيدين (TAs)، وتقارير PDF. التطبيق يتصل بسيرفر Node.js عبر REST + WebSocket للحصول على بيانات لحظية (SESSION_ACTIVATED, GRADE_UPDATED, SEMESTER_CHANGED... إلخ).

1.2 الأدوار (Roles)

الدورالشاشة الرئيسيةالصلاحيات
StudentStudentScreenعرض الجدول، الدرجات، تعديل البروفايل، QR للحضور
DoctorDoctorScreenتفعيل جلسات الحضور، إدارة الدرجات، إدارة المعيدين، التقارير
Teaching Assistant (TA)DoctorScreen (محدود)حسب صلاحيات الـ Doctor المشرف (per-subject permissions)

1.3 الفيتشرز الأساسية

  • Authentication ثلاثي الأوضاع: Student / Doctor / Admin login مع secure token storage.
  • تفعيل جلسة حضور: الدكتور يطلب تشغيل الكاميرا عبر Host API → الكاميرا تكشف الوجوه → الطلاب يأكدون عبر QR personal code.
  • Real-time updates: WebSocket reconnect تلقائي مع heartbeat كل 25 ثانية.
  • GPA Calculator: حساب GPA per-semester / per-level / cumulative مع credit hours.
  • PDF Reports: توليد تقارير حضور كاملة بصور الحالة (Present/Pending/Absent).
  • Offline Cache: أول مرة → API، بعدها cache مدته ساعة عبر SharedPreferences.
  • Server URL Override: المستخدم يقدر يغير IP السيرفر من شاشة Login (مفيد للتطوير المحلي).
  • Theme toggle: Dark / Light / System persisted.
  • Skeleton loading + Shimmer effect أثناء الـ Loading.
  • Face ID + QR Code attendance flow مع timing (مدة الـ confirmation).

6. main.dart — نقطة البداية

الكود

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await ServerConfig.load();                  // (1) تحميل URL السيرفر المحفوظ
  HttpOverrides.global = MyHttpOverrides();   // (2) ثقة بـ SSL للسيرفر المحدد
  await initServiceLocator();                 // (3) تهيئة كل GetIt + Cubits
  runApp(const MyApp());
}

التفسير

  1. ServerConfig.load(): يقرأ من SharedPreferences الـ URL اللي حفظه المستخدم آخر مرة (لو غير الـ default).
  2. MyHttpOverrides: يجعل التطبيق يثق بشهادات SSL لسيرفر معين فقط (للسماح بـ self-signed certs أثناء التطوير).
  3. initServiceLocator: يسجل كل الـ Repositories والـ Cubits في GetIt، ثم يستدعي AuthCubit.init() لاستعادة الجلسة، ثم WebSocketService.connect().

_AuthGate

هو widget داخلي يستمع لـ AuthCubit و:

  • لو المستخدم authenticated وهو doctor أو teaching-assistant → DoctorScreen
  • لو student → StudentScreen
  • لو غير مسجل → LoginScreen

عند تغير الـ user، يستدعي DataCubit.fullReload() تلقائياً لتحديث البيانات بناءً على الـ user الجديد.

ScreenUtilInit

7.11 core/result/result.dart

Sealed class Result<T> يستبدل النمط القديم {'success': bool, 'error': ...}. 

sealed class Result<T> {
  bool get isSuccess;
  T? get valueOrNull;
  AppException? get exceptionOrNull;

  R when<R>({
    required R Function(T) success,
    required R Function(AppException) failure,
  });

  Result<R> map<R>(...);
  Future<Result<R>> flatMap<R>(...);
}

class Success<T> extends Result<T> { final T value; }
class Failure<T> extends Result<T> { final AppException exception; }

المميزات: ما حدش يقدر ينسى يعالج حالة الـ failure (compile-time check بفضل sealed).

8. طبقة State Management (Cubit)

8.1 ليه Cubit وليس BLoC الكامل؟

Cubit أبسط: بدل ما تعمل Event classes لكل action، أنت بتنادي method مباشرة. مناسب للمشروع لأن الـ events بسيطة (login, refresh, updateGrade). BLoC الكامل أفضل لو احتجنا event transformation معقدة (debounce, throttle).

8.2 cubit/shared/loading_state.dart

enum LoadingStatus { initial, loading, loaded, error } + class wrapper LoadingState فيه getters: isLoading, isLoaded, hasError.

8.3 cubit/shared/error_state.dart

Class بسيط بـ hasError, message. (الكود الحديث يستخدم LoadingState.error بدلاً منه).

8.4 cubit/auth/auth_cubit.dart + auth_state.dart

AuthState (Equatable): user, token, isAuthenticated, loadingState + factories: initial / loading / success / error / loggedOut.

AuthCubit methods:

  • init() / checkAuth() — يحاول استعادة الجلسة من SecureStorage. لو نجح: emit success + connect WebSocket.
  • login(username, password, isStudent) — يوحد الـ entry لـ studentLogin / doctorLogin حسب الـ toggle.
  • studentLogin(studentId, password, isFirstLogin) — مع flag للـ first-login (الطالب لازم يغير password).
  • doctorLogin(username, password).
  • updateUserPermissions(newPerms) — يحدث صلاحيات الـ TA في الـ state + يحفظ في SecureStorage.
  • logout() — يمسح الـ token + يقطع الـ WebSocket + emit loggedOut.
  • refreshUserData() — يجيب بيانات الـ user الحديثة من السيرفر (TA يتم تجاوزه عشان بيانات الـ TA بتيجي من اللوجين).

8.5 cubit/data/data_cubit.dart + data_state.dart

DataState: قوائم students/doctors/subjects/lectures/sections/grades/attendance/TAs + currentSemester + currentAcademicYear + loadingState.

Helper methods داخل State: getSubjectsForDoctor, getLecturesForDoctor, getSubjectsForStudent (فلترة بالـ level و department), getSectionsForStudent, getGradesForStudentAndSemester, getAllGradesForStudent, semesterDisplay.

DataCubit methods:

  • init() / loadSystemData() — يحمل semester + academic year الحالي.
  • loadAllData() / fullReload() — يجيب كل البيانات (students, doctors, subjects, lectures, sections, TAs) بالتوازي عبر Future.wait، يفلتر حسب السمستر الحالي، يحفظ في cache.
  • _emitFromCache() — fallback لو الـ network failed.
  • loadStudentGrades / loadStudentGradesWithToken.
  • getGradesForSemester(semester).
  • checkGradesStatus(studentId, token) — debug endpoint.
  • loadAttendance(token).
  • fetchTAsForDoctor(doctorId).
  • fetchTAPermissions / updateTAPermissions.
  • refreshForNewSemester() — يعيد التحميل عند تغير السمستر.
  • updateGrade(updatedGrade) — تحديث grade واحد (يفيد لحظياً عند WebSocket GRADE_UPDATED).
  • clearData() — Reset to initial.

8.6 cubit/theme/theme_cubit.dart

ThemeState فيه ThemeMode (system/light/dark). _loadTheme() يقرأ من SharedPrefs key theme_mode. setTheme(mode) يحفظ + emit. toggleTheme() يقلب بين dark و light.

11. طبقة Services

11.1 SecureStorageService

يستخدم flutter_secure_storage مع Android EncryptedSharedPreferences و iOS Keychain (first_unlock accessibility). فيه helper internal _write/_read/_delete مع error handling (لو القراءة فشلت، يحذف القيمة المعطوبة).

المفاتيح: auth_token_secure, user_data_secure, refresh_token_secure.

الـ Public API: saveToken/getToken/hasToken/deleteToken، saveUserData/getUserData/deleteUserData، saveRefreshToken/getRefreshToken/deleteRefreshToken، clearAll.

11.2 TokenHolder

يلف الـ token في ValueNotifier<String?> ليبقى observable. الفايدة: أي widget يقدر يستمع للتغير بدون polling. فيه authHeader getter يرجع Map<String, String> جاهزة للاستخدام في HTTP requests.

11.3 CacheService

يحفظ students/doctors/subjects/lectures في SharedPrefs مع timestamp. loadAllData(ignoreExpiry: false) ترجع null لو الـ cache أقدم من ساعة. لما الـ network يفشل، الـ DataCubit يستدعيها بـ ignoreExpiry: true لتقديم أحدث snapshot للمستخدم.

11.4 WebSocketService (Singleton)

أهم خدمة في التطبيق للـ realtime. الـ instance واحد عام (WebSocketService.instance) لأن مفيش معنى لفتح أكثر من اتصال.

الـ Streams (broadcast)

الـ Streamالـ Typeمتى يتم البث
dataChangeStreamMapأي DATA_CHANGE
semesterStreamintSEMESTER_CHANGED
academicYearStreamStringACADEMIC_YEAR_CHANGED
gradeUpdateStreamMapGRADE_UPDATED أو DATA_CHANGE entity=grade
sessionActivatedStreamMapSESSION_ACTIVATED أو attendance-session/created
sessionEndedStreamMapSESSION_ENDED أو ended action
reportSavedStreamMapREPORT_SAVED
registrationApprovedStreamListREGISTRATION_APPROVED
levelsPromotedStreamMapLEVELS_PROMOTED
taPermissionsStreamMapteaching-assistant/permissions-updated

الـ Connection lifecycle

  1. connect(): يقرأ token + userData، ينشئ HttpClient بـ badCertificateCallback للسيرفر المعروف، يتصل بـ WebSocket مع timeout 12s.
  2. يبعت REGISTER فوراً مع userId, clientType (mobile-doctor / mobile-student), userRole.
  3. يبدأ heartbeat كل 25 ثانية.
  4. لو الاتصال انقطع: exponential backoff يبدأ من 5 ثواني للوصول لأقصى 30 ثانية، حتى 10 محاولات.
  5. _handleMessage: switch على type للرسائل القياسية + _handleDataChange لرسائل DATA_CHANGE التفصيلية (entity + action).

أنواع الرسائل المدعومة

CONNECTION_ESTABLISHED, REGISTERED, PING (يرد بـ PONG), HEARTBEAT_ACK, DATA_CHANGE, SEMESTER_CHANGED, ACADEMIC_YEAR_CHANGED, SESSION_ACTIVATED, SESSION_ENDED, REPORT_SAVED, GRADE_UPDATED, REGISTRATION_APPROVED, LEVELS_PROMOTED, FULL_SYNC, TOKEN_EXPIRED.

14. تدفق البيانات (Data Flow)

14.1 تسجيل الدخول

LoginScreen
   └─→ AuthCubit.login(username, password, isStudent)
         └─→ AuthRepository.studentLogin / doctorLogin
               └─→ ApiService.studentLogin / doctorLogin (HTTP POST)
                     └─→ Server returns { token, user }
               └─→ SecureStorageService.saveUserData + saveToken
               └─→ ApiService.setToken (in-memory)
         └─→ AuthCubit emit AuthState.success
               └─→ WebSocketService.connect()
   └─→ AuthGate يرى isAuthenticated=true → يعرض DoctorScreen / StudentScreen
   └─→ AuthGate listener → DataCubit.fullReload()

15. ملاحظات تقنية مهمة

15.1 SSL / Self-signed certificates

MyHttpOverrides في main.dart يثق بشهادات SSL للـ host الحالي المضبوط في ServerConfig. ده ضروري لو السيرفر المحلي يستخدم self-signed cert. لو المستخدم غير الـ URL، الـ trust يتحدث تلقائياً.

15.2 Offline-first behavior

DataCubit عند فشل الـ network:

  1. يحاول قراءة الـ cache (CacheService).
  2. لو الـ cache فاضي أو فيه session expired → يعرض error state.
  3. لو الـ cache موجود → يستخدمه (snapshot قديم لكن أفضل من شاشة فارغة).

15.3 Hot restart safety

GetIt registrations كلها بـ isRegistered guard. AppProviders.value يستخدم نفس الـ instances من GetIt بدل ما يعمل جديدة. النتيجة: hot restart بدون تكرار الـ services أو فقدان الـ singletons.

15.4 ScreenUtil + textScaler clamping

استخدام .w / .h / .sp يجعل التصميم responsive. الـ clamping على textScaler [1.0, 1.2] يمنع كسر الـ layout لو المستخدم رافع الـ system font.

15.5 Image cache eviction

بعد رفع avatar جديد، ProfileAvatar.evict(url) يمسح الـ cache في 3 مستويات (cached_network_image, Provider, PaintingBinding) لضمان ظهور الصورة الجديدة فوراً.

15.6 Robust JSON parsing

Section.fromJson يجرب أسماء مفاتيح متعددة (taName, ta_name, teaching_assistant_name...) لأن السيرفر بيرجع أسماء مختلفة في endpoints مختلفة. ده anti-pattern عموماً، لكنه ضروري هنا لأن السيرفر مش under our control.

16. ملخص الـ Workflow للمذاكرة

  1. المستخدم يفتح التطبيق → main.dart → ServerConfig.load → initServiceLocator → AuthCubit.init (يستعيد الجلسة) → DataCubit.init (يحمل البيانات) → WebSocketService.connect.
  2. AuthGate يقرر الشاشة بناءً على authState.user.role.
  3. الشاشة الرئيسية تستخدم BlocBuilder<DataCubit> لعرض البيانات وتسجل StreamSubscriptions على WebSocket streams.
  4. عند تفاعل المستخدم (تسجيل، تعديل، حذف) الشاشة تنادي method على cubit، cubit ينادي repository، repository يستخدم ApiService أو http مباشرة، النتيجة Result<T> ترجع للـ cubit → emit state جديد → BlocBuilder يعيد البناء.
  5. التحديثات اللحظية (من admin/doctors آخرين) تأتي عبر WebSocket → الـ stream المناسب → الشاشة المستمعة تستدعي cubit.refresh() أو cubit.updateX(...).
  6. الـ Offline: لو الـ network فشل، CacheService يقدم آخر snapshot. لو نجح الـ network، يتم تحديث الـ cache.
  7. الـ Token: محفوظ في SecureStorage (مشفر hardware-backed). ApiService يحتفظ بنسخة in-memory للسرعة. عند logout، الاتنين يتمسحوا.

17. الخلاصة

المشروع يستخدم memorable architecture: UI ← Cubit ← Repository ← Service ← API. كل طبقة لها مسؤولية واضحة وقابلة للاختبار منفصلة. الـ packages المختارة مناسبة لحجم ومتطلبات المشروع بدون over-engineering. الـ realtime عبر WebSocket مع reconnect logic متقن، والـ offline-first cache يحسن تجربة المستخدم في الـ scenarios الضعيفة. الـ Models immutable مع copyWith، والـ Result<T> يفرض معالجة الأخطاء على مستوى الـ compiler.
